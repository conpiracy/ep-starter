pi can create themes. Ask it to build one for your setup.

Themes are JSON files that define colors for the TUI.

          Locations
        
        
          
          
          Copied
        
        
Pi loads themes from:

Built-in: dark, light
Global: ~/.pi/agent/themes/*.json
Project: .pi/themes/*.json (only after the project is trusted)
Packages: themes/ directories or pi.themes entries in package.json
Settings: themes array with files or directories
CLI: --theme &lt;path&gt; (repeatable)

Disable discovery with --no-themes.

        
          Selecting a Theme
        
        
          
          
          Copied
        
        
Select a theme via /settings or in settings.json:
{
  &quot;theme&quot;: &quot;my-theme&quot;
}

On first run, pi detects your terminal background and defaults to dark or light.

          Theme Format
        
        
          
          
          Copied
        
        
{
  &quot;$schema&quot;: &quot;https://raw.githubusercontent.com/earendil-works/pi/main/packages/coding-agent/src/modes/interactive/theme/theme-schema.json&quot;,
  &quot;name&quot;: &quot;my-theme&quot;,
  &quot;vars&quot;: {
    &quot;blue&quot;: &quot;#0066cc&quot;,
    &quot;gray&quot;: 242
  },
  &quot;colors&quot;: {
    &quot;accent&quot;: &quot;blue&quot;,
    &quot;muted&quot;: &quot;gray&quot;,
    &quot;text&quot;: &quot;&quot;,
    ...
  }
}

name is required, must be unique, and must not contain /.
vars is optional. Define reusable colors here, then reference them in colors.
colors must define all 51 required tokens. thinkingMax is optional and falls back to thinkingXhigh.

The $schema field enables editor auto-completion and validation.
