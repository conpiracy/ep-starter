          Copied
        
        
Radius is a dynamic pi-messages gateway. /login radius stores OAuth tokens in auth.json; the gateway catalog is refreshed independently and cached in models-store.json. Custom Radius gateways can be declared in models.json with &quot;oauth&quot;: &quot;radius&quot; and a gateway baseUrl.

          Auth File
        
        
          
          
          Copied
        
        
Store credentials in ~/.pi/agent/auth.json:
{
  &quot;anthropic&quot;: { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;sk-ant-...&quot; },
  &quot;ant-ling&quot;: { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;...&quot; },
  &quot;openai&quot;: { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;sk-...&quot; },
  &quot;deepseek&quot;: { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;sk-...&quot; },
  &quot;nvidia&quot;: { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;nvapi-...&quot; },
  &quot;google&quot;: { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;...&quot; },
  &quot;opencode&quot;: { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;...&quot; },
  &quot;opencode-go&quot;: { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;...&quot; },
  &quot;together&quot;: { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;...&quot; },
  &quot;xiaomi&quot;: { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;...&quot; },
  &quot;xiaomi-token-plan-cn&quot;:  { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;...&quot; },
  &quot;xiaomi-token-plan-ams&quot;: { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;...&quot; },
  &quot;xiaomi-token-plan-sgp&quot;: { &quot;type&quot;: &quot;api_key&quot;, &quot;key&quot;: &quot;...&quot; }
}

The file is created with 0600 permissions (user read/write only). Auth file credentials take priority over environment variables.
API key credentials can also include provider-scoped environment values. These values are used before process environment variables when resolving the credential key, provider/model headers, and provider configuration such as Cloudflare account IDs, Azure OpenAI settings, Vertex project/location, Bedrock settings, PI_CACHE_RETENTION, and HTTP_PROXY/HTTPS_PROXY.
