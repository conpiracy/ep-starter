operation of the coding agent via a JSON protocol over stdin/stdout. This is useful for embedding the agent in other applications, IDEs, or custom UIs.
Note for Node.js/TypeScript users: If you&#39;re building a Node.js application, consider using AgentSession directly from @earendil-works/pi-coding-agent instead of spawning a subprocess. See src/core/agent-session.ts for the API. For a subprocess-based TypeScript client, see src/modes/rpc/rpc-client.ts.

        
          Starting RPC Mode
        
        
          
          
          Copied
        
        
pi --mode rpc [options]

Common options:

--provider &lt;name&gt;: Set the LLM provider (anthropic, openai, google, etc.)
--model &lt;pattern&gt;: Model pattern or ID (supports provider/id and optional :&lt;thinking&gt;)
--name &lt;name&gt; / -n &lt;name&gt;: Set the session display name at startup
--no-session: Disable session persistence
--session-dir &lt;path&gt;: Custom session storage directory

        
          Protocol Overview
        
        
          
          
          Copied
        
        

Commands: JSON objects sent to stdin, one per line
Responses: JSON objects with type: &quot;response&quot; indicating command success/failure
Events: Agent events streamed to stdout as JSON lines

All commands support an optional id field for request/response correlation. If provided, the corresponding response will include the same id.

        
          Framing
        
        
          
          
          Copied
        
        
RPC mode uses strict JSONL semantics with LF (\n) as the only record delimiter.
This matters for clients:

Split records on \n only
Accept optional \r\n input by stripping a trailing \r
Do not use generic line readers that treat Unicode separators as newlines

In particular, Node readline is not protocol-compliant for RPC mode because it also splits on U+2028 and U+2029, which are valid inside JSON strings.

          Copied
        
        

        
          Prompting
        
        
          
          
          Copied
        
        

        
          prompt
        
        
          
          
          Copied
        
        
Send a user prompt to the agent. The command response is emitted after the prompt is accepted, queued, or handled. Events continue streaming asynchronously after acceptance.
{&quot;id&quot;: &quot;req-1&quot;, &quot;type&quot;: &quot;prompt&quot;, &quot;message&quot;: &quot;Hello, world!&quot;}

With images:
{&quot;type&quot;: &quot;prompt&quot;, &quot;message&quot;: &quot;What&#39;s in this image?&quot;, &quot;images&quot;: [{&quot;type&quot;: &quot;image&quot;, &quot;data&quot;: &quot;base64-encoded-data&quot;, &quot;mimeType&quot;: &quot;image/png&quot;}]}

During streaming: If the agent is already streaming, you must specify streamingBehavior to queue the message:
{&quot;type&quot;: &quot;prompt&quot;, &quot;message&quot;: &quot;New instruction&quot;, &quot;streamingBehavior&quot;: &quot;steer&quot;}

&quot;steer&quot;: Queue the message while the agent is running. It is delivered after the current assistant turn finishes executing its tool calls, before the next LLM call.
&quot;followUp&quot;: Wait until the agent finishes. Message is delivered only when agent stops.

If the agent is streaming and no streamingBehavior is specified, the command returns an error.
Extension commands: If the message is an extension command (e.g., /mycommand), it executes immediately even during streaming. Extension commands manage their own LLM interaction via pi.sendMessage().
Input expansion: Skill commands (/skill:name) and prompt templates (/template) are expanded before sending/queueing.
Response:
{&quot;id&quot;: &quot;req-1&quot;, &quot;type&quot;: &quot;response&quot;, &quot;command&quot;: &quot;prompt&quot;, &quot;success&quot;: true}

success: true means the prompt was accepted, queued, or handled immediately. success: false means the prompt was rejected before acceptance. Failures after acceptance are reported through the normal event and message stream, not as a second response for the same request id.
The images field is optional. Each image uses ImageContent format: {&quot;type&quot;: &quot;image&quot;, &quot;data&quot;: &quot;base64-encoded-data&quot;, &quot;mimeType&quot;: &quot;image/png&quot;}.

        
          steer
        
        
          
          
          Copied
        
        
Queue a steering message while the agent is running. It is delivered after the current assistant turn finishes executing its tool calls, before the next LLM call. Skill commands and prompt templates are expanded. Extension commands are not allowed (use prompt instead).
{&quot;type&quot;: &quot;steer&quot;, &quot;message&quot;: &quot;Stop and do this instead&quot;}

With images:
{&quot;type&quot;: &quot;steer&quot;, &quot;message&quot;: &quot;Look at this instead&quot;, &quot;images&quot;: [{&quot;type&quot;: &quot;image&quot;, &quot;data&quot;: &quot;base64-encoded-data&quot;, &quot;mimeType&quot;: &quot;image/png&quot;}]}

The images field is optional. Each image uses ImageContent format (same as prompt).
Response:
{&quot;type&quot;: &quot;response&quot;, &quot;command&quot;: &quot;steer&quot;, &quot;success&quot;: true}

See set_steering_mode for controlling how steering messages are processed.

        
          follow_up
        
        
          
          
          Copied
        
        
Queue a follow-up message to be processed after the agent finishes. Delivered only when agent has no more tool calls or steering messages. Skill commands and prompt templates are expanded. Extension commands are not allowed (use prompt instead).
{&quot;type&quot;: &quot;follow_up&quot;, &quot;message&quot;: &quot;After you&#39;re done, also do this&quot;}

With images:
{&quot;type&quot;: &quot;follow_up&quot;, &quot;message&quot;: &quot;Also check this image&quot;, &quot;images&quot;: [{&quot;type&quot;: &quot;image&quot;, &quot;data&quot;: &quot;base64-encoded-data&quot;, &quot;mimeType&quot;: &quot;image/png&quot;}]}

The images field is optional. Each image uses ImageContent format (same as prompt).
Response:
{&quot;type&quot;: &quot;response&quot;, &quot;command&quot;: &quot;follow_up&quot;, &quot;success&quot;: true}

See set_follow_up_mode for controlling how follow-up messages are processed.

        
          abort
        
        
          
          
          Copied
        
        
Abort the current agent operation.
{&quot;type&quot;: &quot;abort&quot;}

Response:
{&quot;type&quot;: &quot;response&quot;, &quot;command&quot;: &quot;abort&quot;, &quot;success&quot;: true}

        
          new_session
        
        
          
          
          Copied
        
        
Start a fresh session. Can be cancelled by a session_before_switch extension event handler.
{&quot;type&quot;: &quot;new_session&quot;}

With optional parent session tracking:
{&quot;type&quot;: &quot;new_session&quot;, &quot;parentSession&quot;: &quot;/path/to/parent-session.jsonl&quot;}

Response:
{&quot;type&quot;: &quot;response&quot;, &quot;command&quot;: &quot;new_session&quot;, &quot;success&quot;: true, &quot;data&quot;: {&quot;cancelled&quot;: false}}

If an extension cancelled:
{&quot;type&quot;: &quot;response&quot;, &quot;command&quot;: &quot;new_session&quot;, &quot;success&quot;: true, &quot;data&quot;: {&quot;cancelled&quot;: true}}

{&quot;type&quot;: &quot;get_state&quot;}

Response:
{
  &quot;type&quot;: &quot;response&quot;,
  &quot;command&quot;: &quot;get_state&quot;,
  &quot;success&quot;: true,
  &quot;data&quot;: {
    &quot;model&quot;: {...},
    &quot;thinkingLevel&quot;: &quot;medium&quot;,
    &quot;isStreaming&quot;: false,
    &quot;isCompacting&quot;: false,
    &quot;steeringMode&quot;: &quot;all&quot;,
    &quot;followUpMode&quot;: &quot;one-at-a-time&quot;,
    &quot;sessionFile&quot;: &quot;/path/to/session.jsonl&quot;,
    &quot;sessionId&quot;: &quot;abc123&quot;,
    &quot;sessionName&quot;: &quot;my-feature-work&quot;,
    &quot;autoCompactionEnabled&quot;: true,
    &quot;messageCount&quot;: 5,
    &quot;pendingMessageCount&quot;: 0

}

The model field is a full Model object or null. The sessionName field is the display name set via set_session_name, or omitted if not set.

{&quot;type&quot;: &quot;get_messages&quot;}

Response:
