Review the following code for:

1. **Correctness** — bugs, edge cases, race conditions
2. **Security** — injection, auth, data exposure, input validation
3. **Performance** — bottlenecks, unnecessary allocations, caching
4. **Style** — idiomatic patterns, readability, consistency

Be specific and actionable. Prioritize correctness and security.
